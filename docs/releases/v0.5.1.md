# szn v0.5.1

**7 commits** since v0.5.0.

---

## Headline: Event Loop PTY FD Leaks & Memory Safety Fixes

This maintenance release focuses on critical stability, memory safety, and resource leak fixes:

- **UAF Segfault on Startup** — Fixed a use-after-free segfault on daemon startup where the daemon attempted to spawn and initialize the `"default"` session's pane shell even if it was killed by a client handshake within the initial 16ms window.
- **PTY FD and Memory Leaks** — Resolved multiple resource leaks where closed PTY master descriptors remained in the event loop and pane state was not deinitialized on session, window, or individual pane destruction (e.g., via `kill-pane` or event loop triggers).
- **Szn Startup Freeze** — Fixed a bug where the server failed to launch the default shell, freezing on startup. The daemon was hardcoded to look for a default session ID of `0`, but session IDs start at `1`.
- **Pane Boundary & Coordinate Calculations** — Refactored pane boundary checks and cursor coordinate shifts during terminal resizing and rendering.
