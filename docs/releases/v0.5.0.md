# szn v0.5.0

**16 commits** since v0.4.2.

---

## Headline: Sixel Graphics Support via Grid Allocation & Registry Model

This minor release introduces proper sixel graphics rendering built on a new Grid Allocation & Registry Model. Sixel images are now allocated to a deterministic grid, contained within their cell boundaries, and force-cleared only when genuinely overlapped — eliminating smearing and image loss across scrolling, copy-mode, and scrollback.

- **Sixel Grid Allocation & Registry Model** — Replaced the previous ad-hoc sixel handling with a grid-based allocation and registry. Images are anchored once on creation, shifted on scroll, and boundary-contained so they never bleed past their cell region. A force-clear is triggered only when an overlapping sixel is actually removed.
- **Configurable Pixel↔Cell Conversion** — Sixel pixel-to-cell conversion now uses a configurable cell size instead of a fixed assumption, so images align correctly on terminals with non-default font metrics.
- **On-Demand Cell Size** — The terminal's cell size is queried when a sixel is displayed (via `cell_size`) rather than at startup, and sixels are buffered until the cell size is known. Oversized sixels are dropped instead of corrupting the grid.

---

## New Features

- **Sixel Grid Allocation & Registry Model** — Deterministic grid allocation with boundary containment and force-clear logic for correct sixel rendering.
- **Configurable sixel cell size** — Pixel↔cell conversion honors the terminal's actual cell dimensions.
- **Sixel buffering until cell size is measured** — Sixels are deferred until the display client reports `cell_size`, preventing misplaced or oversized images.

## Bug Fixes

| Description | Severity |
|-------------|----------|
| Sixel overlay smeared on scroll/move instead of being erased | High |
| Copy-mode / scrollback sixel loss after the registry wraps | High |
| Partially-scrolled sixel images lost instead of kept visible | Medium |
| Sixel anchor re-computed each render, drifting on scroll | Medium |
| Per-pane sixel dedupe done globally, dropping valid images | Medium |
| `force_clear` only honored from active pane in `renderAll` | Medium |
| `force_clear` triggered for any removal, not just overlaps | Low |
| Sixel pixel↔cell conversion used fixed cell size | Low |
| Sixels rendered before cell size known / oversized sixels | Low |
