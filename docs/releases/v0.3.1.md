# szn v0.3.1

**9 commits** since v0.3.0.

---

## Headline: Robust Copy/Paste, Non-Blocking PTY, and Unicode Combining Marks Preservation

This patch release resolves critical bugs in copy-paste mechanics, terminal I/O deadlock freezes, memory safety, and Thai vowel representation:

- **Unicode Combining Character Copying** ‚ÄĒ Fixes a major bug where copying Thai text in copy-mode omitted upper/lower vowels (`ŗłī`, `ŗłĶ`, `ŗł∂`, `ŗłł`, `ŗłĻ`) and tone marks (`ŗĻą`, `ŗĻČ`). The yank logic now parses cell combining indices and encodes them as UTF-8 in the yanked buffer.
- **Copy-mode Soft-wrap Merging** ‚ÄĒ Automatically merges soft-wrapped rows into a single logical line when yanking, trimming trailing empty cells and preventing unwanted newlines in the middle of copied sentences.
- **Auto-Reflow on Copy Mode Entry** ‚ÄĒ Entering copy-mode (via keyboard shortcut or mouse scroll-up) now automatically triggers a pane reflow first, ensuring screen row wrapping attributes are cleanly mapped based on current layout width before selection.
- **PTY-based Input Writing (Real Paste)** ‚ÄĒ Refactored `paste-buffer` and `send-keys` to write directly to the PTY stdin stream (`pty.writeInput`) rather than drawing directly to the screen grid. Pasting text now correctly feeds input to the running program/shell.
- **Non-blocking PTY & Freeze Prevention** ‚ÄĒ Sets the master PTY descriptor to non-blocking and implements a draining write helper. During large pastes, if the PTY stdin buffer becomes full, `szn` will drain the PTY stdout to unblock the child shell, preventing bidirectional deadlock hangs.
- **Stack-Overflow & Memory Safety Fixes** ‚ÄĒ Converts recursive layout node functions (`deinitNode` and `findFirstLeaf`) to iterative implementations to prevent stack overflows on complex split hierarchies, and null-terminates wide-character buffers passed to `libthai` to prevent out-of-bounds reads.

## Bug Fixes

| Description | Severity |
|-------------|----------|
| PTY write lockup / server freezes when pasting large blocks of text | Critical |
| Yanking text ignores cell combining characters (dropping Thai vowels & tone marks) | Critical |
| `paste-buffer` and `send-keys` write to screen grid instead of active PTY stdin | High |
| Yanking soft-wrapped text leaves newlines and trailing spaces in the clipboard | Medium |
| Recursive layout node cleanup can stack overflow on deeply split structures | Medium |
| Out-of-bounds read in `findWordBreaks` due to missing null-terminator for libthai | Medium |
| CJK wide characters yanked with extra trailing spacer cells | Low |
