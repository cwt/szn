# szn v0.4.2

**18 commits** since v0.4.1.

---

## Headline: IPC Protocol Compliance Audit, Zig 0.16 Modernization, and Bracketed Paste

This patch release hardens the client/server IPC protocol, modernizes the codebase for Zig 0.16.0, and adds bracketed paste handling. No new user-facing commands were introduced.

- **IPC Protocol Compliance Audit (BUGS #186–#193)** — A deep-dive into the client/server protocol surfaced eight issues, all now fixed: dead `IdentifyTerm` struct on the wire (sends a raw string instead), unimplemented reserved message types removed, single-session attach restriction documented, protocol struct packing corrected with layout tests, consistent packet size limits enforced across parsers, unrecognized message types now logged (instead of silently dropped), `Packet.deserialize` made safe for buffers with trailing data, and sixel pixel dimensions now parsed from raster attributes in DCS.
- **Zig 0.16.0 Modernization** — Removed the deprecated `@cImport` in favor of native C externs, and renamed `std.ArrayListUnmanaged` to `std.ArrayList` (the unmanaged type in 0.16.0). Stray backup files were also cleaned up.
- **Bracketed Paste Mode** — Added bracketed paste mode support: the terminal mode is toggled via `Term`, paste execution is wrapped in bracketed-paste boundaries when enabled, and display clients track the mode for correct alt-screen/normal-screen rendering.

## Bug Fixes

| Description | Severity |
|-------------|----------|
| Paste buffers larger than 16 KB could hang the server | High |
| IPC `Packet.deserialize` unsafe for streams with trailing data | High |
| Inconsistent packet size limits across protocol parsers | Medium |
| Sixel dimensions unknown, cursor advance only approximated | Medium |
| Protocol structs documented as packed but not actually packed | Low |
| Unrecognized message types silently dropped on client and server | Low |
| Reserved/unimplemented message types present in protocol enum | Low |
| Dead `IdentifyTerm` struct on the wire | Low |
